## How I approach each problem

I don't just write the first solution that comes to mind. My usual process is:

1. Read the problem fully, understand the constraints
2. Think of a brute-force approach first — just to get an idea of the problem structure
3. Look for patterns (sliding window? binary search? DP?)
4. Code the optimal solution
5. Dry run it on edge cases before submitting
6. If I get it wrong, I go back and understand *why* before moving on

Some problems have comments in the code explaining my thought process — especially the ones that took me a while to figure out.
